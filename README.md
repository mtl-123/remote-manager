## 💡 项目初衷

在日常运维、多服务器管理、办公远程桌面场景中，长期面临以下痛点：

1. **工具碎片化**

    - Windows 远程桌面、FreeRDP、Xrdp、原生 SSH、FinalShell、Xshell 等工具混用；

    - 不同协议（RDP/SSH）需要打开多个软件，切换繁琐。

2. **主机无统一管理**

    - 主机IP、端口、账号、密码分散记录，无统一配置库；

    - 无分组、无快速检索、无最近连接记录，多主机运维效率极低。

3. **Linux 下 RDP 体验差**

    - Linux 桌面环境缺少轻量化、稳定的 RDP 客户端；

    - FreeRDP 原生命令参数复杂，记忆成本高，无法保存主机配置。

4. **文件传输割裂**

    - SSH 文件传输依赖零散 `scp` 命令，无统一交互入口；

    - 缺乏进度条、批量传输、远程一键部署工具能力。

5. **安全与轻量化需求**

    - 闭源远程工具存在隐私泄露、后门风险；

    - 想要**单文件、无GUI、纯终端、低资源、可离线运行**的运维工具。

### 开发本项目核心目的

- 用 **Rust** 编写，高性能、内存安全、无GC、跨平台；

- 整合 **RDP + SSH** 两大核心远程协议，一个工具全部搞定；

- 本地加密存储主机密码、配置文件本地化，保护运维资产；

- 纯终端交互式操作，适配服务器无桌面、极简Linux环境；

- 一键编译、单文件分发，无复杂依赖，适合内网/服务器部署。

## 📌 项目介绍

`remote-manager` 是一款**纯终端交互式远程连接管理器**，专为 Linux 运维工程师、IT 管理员、多设备办公人群设计。

整合 `FreeRDP3` + `OpenSSH` + `trzsz` 文件传输能力，实现：

- Windows RDP 远程桌面连接（Linux 完美适配）

- Linux SSH 远程终端连接

- 统一主机配置管理、加密密码存储

- 批量主机检测、批量命令执行

- 一体化 SCP 文件上传/下载

- 极简菜单交互，傻瓜式操作，零命令记忆成本

## 📦 内置trzsz工具（关键）

本工具**内置 trzsz 高速文件传输二进制**，无需手动安装，程序运行时会自动释放到本地缓存目录，支撑 SCP 文件传输、断点续传功能。

### 1. 资源来源

- trzsz 官方发布版本：v1.2.0

- 官方下载地址：https://github.com/trzsz/trzsz-go/releases/download/v1.2.0/trzsz_1.2.0_linux_x86_64.tar.gz

- 适配平台：linux_x86_64（兼容所有主流Linux发行版）

### 2. 内置方式

项目将 trzsz 官方二进制文件转换为 **base64 编码文件**，编译时直接打包进程序，避免运行时依赖外部文件：

- `trz.b64`：trz 上传工具（本地→远程）

- `tsz.b64`：tsz 下载工具（远程→本地）

- `trzsz.b64`：trzsz 核心客户端（整合上传下载）

### 3. 一键生成脚本（build_trzsz_b64.sh）

项目根目录已提供一键脚本，自动下载官方二进制、生成 base64 文件，内容如下（可直接复制创建）：

```bash
#!/bin/bash

# 下载 trzsz 官方二进制包（v1.2.0 linux_x86_64）
wget https://github.com/trzsz/trzsz-go/releases/download/v1.2.0/trzsz_1.2.0_linux_x86_64.tar.gz -O trzsz.tar.gz

# 解压压缩包（自动生成 trzsz_1.2.0_linux_x86_64 目录）
tar -zxvf trzsz.tar.gz

# 进入解压目录，提取核心二进制文件
cd trzsz_1.2.0_linux_x86_64

# 生成 base64 文件（-w 0 表示不换行，适配Rust include_bytes读取）
base64 -w 0 trz > ../trz.b64
base64 -w 0 tsz > ../tsz.b64
base64 -w 0 trzsz > ../trzsz.b64

# 回到项目根目录
cd ..

# 清理临时文件（解压目录、压缩包），只保留b64文件
rm -rf trzsz.tar.gz trzsz_1.2.0_linux_x86_64

# 提示完成
echo "✅ trzsz base64 文件生成完成！"
echo "📄 生成文件：trz.b64  tsz.b64  trzsz.b64"
echo "💡 请确保这三个文件与 src 目录同级，否则编译会失败"
```

### 4. 运行时自动释放

程序首次运行时，会自动将 base64 编码解码为可执行二进制文件，释放到以下目录，并自动赋予 0o755 可执行权限：

```bash
remote-manager-data/cache/
```

后续运行无需重复释放，直接复用缓存文件，提升启动速度。

### 5. 更新 trzsz 版本

若需要更新 trzsz 版本，只需修改脚本中的 `wget` 下载地址（替换为对应版本的官方链接），重新执行脚本即可覆盖原有 b64 文件，再重新编译项目即可。

## 🎯 完整使用案例

以下是最常用的「添加主机→连接→文件传输→批量检测」完整流程，适配日常运维场景。

### 场景：添加内网 Windows 办公主机 + Linux 服务器，完成远程连接与文件传输

#### 1. 启动程序

编译完成后，进入产物目录，直接执行二进制文件：

```bash
./remote-manager
```

启动后展示交互式主菜单（清晰易懂，无需记忆命令）：

```bash
===================== remote-manager =====================
1. 连接 RDP 主机
2. 连接 SSH 主机
3. 添加新主机
4. 编辑主机
5. 删除主机
6. SCP 文件传输
7. 批量执行命令
8. 配置管理
9. 批量检测主机连通性
0. 退出
=========================================================
请选择 (0-9):
```

#### 2. 添加 Windows RDP 主机（办公机）

1. 主菜单输入 `3`，选择「添加新主机」；

2. 类型选择 `1`（RDP，对应 Windows 远程桌面）；

3. 主机名称：输入 `Win10-办公机`（自定义，便于识别）；

4. IP 地址：输入 `10`（智能补全为 192.168.1.10，内网场景无需输入完整IP）；

5. 端口：默认 `3389`（RDP 标准端口，直接回车即可）；

6. 用户名：默认 `Administrator`（Windows 管理员账号，可修改为自己的账号）；

7. 密码：输入 Windows 登录密码（输入时不显示明文，自动AES加密保存）；

8. 本地共享目录：直接回车（默认使用当前用户家目录，远程可访问本地文件）；

9. 提示「✅ 主机添加成功！」，完成 RDP 主机添加。

#### 3. 添加 Linux SSH 服务器

1. 主菜单输入 `3`，再次选择「添加新主机」；

2. 类型选择 `2`（SSH，对应 Linux 服务器）；

3. 主机名称：输入 `Linux-运维服务器`；

4. IP 地址：输入 `192.168.1.20`（完整IP，也可输入 `1.20` 自动补全）；

5. 端口：输入 `50022`（自定义 SSH 端口，默认 50022，可修改）；

6. 用户名：输入 `root`（服务器管理员账号）；

7. 密码：输入服务器登录密码（留空则使用密钥认证）；

8. SSH 密钥：直接回车（默认使用 ~/.ssh/id_ed25519，可自定义密钥路径）；

9. 提示「✅ 主机添加成功！」，完成 SSH 主机添加。

#### 4. 连接 RDP 远程桌面（Windows 办公机）

1. 主菜单输入 `1`，选择「连接 RDP 主机」；

2. 展示已添加的 RDP 主机列表，输入对应序号（如 `1`）；

3. 选择 RDP 功能：输入 `1 2`（勾选「全屏模式」+「剪切板同步」）；

4. 程序自动拉起 xfreerdp3 远程桌面，无缝操作 Windows 办公机；

5. 退出远程桌面：按 `Ctrl + Alt + Enter` 退出全屏，关闭窗口即可。

#### 5. 连接 SSH 服务器（Linux 运维）

1. 主菜单输入 `2`，选择「连接 SSH 主机」；

2. 展示已添加的 SSH 主机列表，输入对应序号（如 `2`）；

3. 若服务器未安装 trzsz，程序自动部署（提示「📦 部署 trzsz 工具...」）；

4. 自动连接 SSH 终端，提示「🔌 连接中... Ctrl+] 退出」；

5. 正常执行运维命令（如 `ls -l`、`top`），支持 trzsz 上传下载。

#### 6. SCP 文件传输（本地 ↔ 远程）

1. 主菜单输入 `6`，选择「SCP 文件传输」；

2. 选择需要传输的 SSH 主机（输入对应序号）；

3. 选择传输方向：输入 `1`（上传，本地→远程）；

4. 输入本地路径：`/home/user/test.txt`（需要传输的本地文件）；

5. 输入远程路径：直接回车（默认 ~/，即服务器 root 家目录）；

6. 若安装了 pv，会显示传输进度条（如「100% 0.5MB/0.5MB」）；

7. 提示「✅ 传输成功」，完成文件上传；

8. 下载操作类似：选择方向 `2`，输入远程路径和本地保存路径即可。

#### 7. 批量检测所有主机连通性

1. 主菜单输入 `9`，选择「批量检测主机连通性」；

2. 程序自动扫描所有已添加主机的 IP+端口，超时时间 2 秒；

3. 检测完成后输出统计结果，示例：

```bash
🚀 批量检测主机TCP端口连通性（超时2秒）
------------------------------------------------------------
检测 192.168.1.10 (RDP)...✅ 成功
检测 192.168.1.20 (SSH)...✅ 成功
------------------------------------------------------------
📊 检测完成：
总主机数：2
✅ 连通成功：2 个
❌ 连通失败：0 个

✅ 成功IP：192.168.1.10 | 192.168.1.20
```

#### 8. 退出程序

主菜单输入 `0`，提示「👋 再见！」，正常退出程序，所有配置自动保存。

## 📂 数据目录结构

程序自动在二进制文件同级目录生成隔离数据文件夹，**绿色免安装**，迁移时直接拷贝即可：

```bash
remote-manager-data/
├── config/          # 配置目录
│   └── config.yaml  # 主机配置文件（密码AES加密存储）
│   └── config.yaml.bak  # 自动配置备份（防止配置损坏）
├── ssh/             # SSH 指纹目录
│   └── known_hosts  # SSH 主机指纹记录（避免重复确认）
├── cache/           # 工具缓存目录
│   ├── trz          # 自动释放的 trz 上传工具
│   ├── tsz          # 自动释放的 tsz 下载工具
│   └── trzsz        # 自动释放的 trzsz 核心工具
└── logs/            # 运行日志目录
    └── remote-manager.log  # 操作日志（便于排查问题）
```

### 关键说明

- 迁移设备：直接拷贝 `remote-manager` 二进制 + `remote-manager-data` 文件夹，即可在新机器上恢复所有配置；

- 重置数据：删除 `remote-manager-data` 文件夹，程序下次启动会自动生成新的空白配置；

- 配置备份：程序每次修改配置都会自动生成 `config.yaml.bak`，配置损坏时可手动替换恢复。

## 🛡️ 安全设计

1. **密码加密存储**

    - 所有主机密码采用 `AES-256-GCM` 对称加密算法存储，明文不落地、不打印、不日志记录；

    - 加密密钥内置（可自行修改源码中的 `ENCRYPTION_KEY`），进一步提升安全性。

2. **文件权限管控**

    - 配置文件（`config.yaml`）、日志文件权限设置为 0o600，仅当前用户可读写；

    - 数据目录（`remote-manager-data`）权限设置为 0o755，仅当前用户可修改，其他用户只读。

3. **网络安全**

    - RDP 连接默认忽略无效证书，避免频繁弹窗，同时保证连接安全；

    - SSH 连接严格校验主机指纹，存储在 `known_hosts`，防止中间人攻击；

    - 所有网络连接设置超时控制，避免程序卡死。

4. **密钥安全**

    - SSH 私钥路径自定义，不强制存储密钥文件，遵循最小权限原则；

    - 密钥文件仅在连接时读取，不缓存、不存储，避免密钥泄露。

## ❌ 常见问题与排查

### 1. 编译报错：could not find `trz.b64` / `tsz.b64` / `trzsz.b64`

原因：缺少 trzsz 对应的 base64 文件，未执行一键生成脚本。

解决：执行项目根目录的 `./build_trzsz_b64.sh`，生成 3 个 b64 文件后重新编译。

### 2. 运行报错：未找到 xfreerdp3 / sshpass / pv

原因：系统缺少运行依赖，程序首次启动未引导安装（或安装失败）。

解决：手动安装依赖：

```bash
sudo apt update
sudo apt install -y freerdp3-x11 openssh-client sshpass coreutils pv
```

### 3. SSH 连接失败：status code 255

原因：IP、端口、密码错误，或密钥路径不正确、密钥无权限。

解决：

- 检查主机 IP、端口是否正确（主菜单选择 4 编辑主机）；

- 确认密码正确（重新编辑主机，更新密码）；

- 密钥登录：检查密钥路径是否正确，密钥文件权限设置为 0o600（`chmod 600 ~/.ssh/id_ed25519`）。

### 4. SCP 传输失败：无进度条、提示权限不足

原因：未安装 pv，或远程路径无写入权限。

解决：

- 安装 pv：`sudo apt install pv`；

- 远程路径选择有写入权限的目录（如 ~/、/tmp/）。

### 5. 配置文件损坏，程序无法启动

原因：异常退出（如断电、强制杀死进程）导致配置文件损坏。

解决：替换配置备份文件：

```bash
cd remote-manager-data/config
cp config.yaml.bak config.yaml
```

重新启动程序即可恢复配置。
